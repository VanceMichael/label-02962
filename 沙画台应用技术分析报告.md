# 沙画台模拟应用技术分析报告

## 1. 架构分析

### 1.1 项目分层架构及职责

项目采用典型的MVC（Model-View-Controller）分层架构，清晰分离了数据模型、界面展示和业务逻辑：

| 层级 | 目录位置 | 职责描述 |
|------|----------|----------|
| **模型层 (Model)** | `/backend/models/` | 定义核心数据结构和业务实体<br>- `particle.py`: 沙粒粒子数据模型，使用`dataclass(slots=True)`优化性能<br>- `brush.py`: 画笔类型枚举，统一管理所有画笔工具 |
| **视图层 (View)** | `/backend/windows/` `/backend/widgets/` | 负责界面渲染和用户交互<br>- `main_window.py`: 主窗口，实现工具栏、控制面板和画布布局<br>- `canvas.py`: 沙画画布组件，核心绘制和粒子系统实现<br>- `tool_button.py`: 自定义工具按钮组件 |
| **工具层 (Utils)** | `/backend/utils/` | 提供通用工具函数<br>- `validators.py`: 参数校验模块，统一处理输入验证和错误处理<br>- `logger.py`: 日志系统配置 |
| **入口层** | `/backend/main.py` | 程序入口，负责初始化日志、Qt应用和主窗口 |

### 1.2 画布组件(Canvas)绘制和粒子效果实现

画布组件`SandCanvas`继承自`QWidget`，是应用的核心渲染组件：

#### 沙粒绘制实现
1. **双画布设计**：
   - `canvas_image`(ARGB32格式)：存储最终显示的沙画图像
   - `sand_density`(Grayscale8格式)：存储沙子密度信息（255表示满沙，0表示无沙）
2. **绘制流程**：
   - 鼠标事件触发绘制操作，根据画笔类型调用对应的绘制方法
   - 使用`QPainter`进行绘制，支持径向渐变、抗锯齿等效果
   - 绘制完成后保存历史记录，支持撤销/重做功能

#### 粒子效果实现
- 采用独立的`QTimer`驱动粒子系统，每30ms更新一次粒子状态
- 粒子绘制采用**颜色分组优化**，按粒子颜色rgba值分组，减少QPainter状态切换次数
- 粒子物理更新采用**单次遍历+原地过滤**算法，避免O(n²)删除操作

## 2. 核心算法

### 2.1 粒子系统实现原理

粒子系统通过`SandParticle`数据模型和`_spawn_particles`、`_update_particles`两个核心方法实现：

#### 粒子生成
```python
def _spawn_particles(self, pt: QPoint, count: int, strong: bool = False, spray: bool = False):
```
- **触发时机**：每次绘制操作时根据画笔类型生成不同数量的粒子
- **初始化参数**：随机生成粒子的位置、速度、大小、颜色和生命周期
- **数量控制**：设置`max_particles=500`上限，防止粒子无限增长导致性能下降
- **差异化生成**：不同画笔生成的粒子速度和数量不同，拳头画笔速度更快，喷洒画笔速度最高

#### 粒子运动
```python
def _update_particles(self):
```
- **物理模拟**：每个粒子受重力和摩擦系数影响，更新速度和位置
- **运动方程**：
  ```
  vy = (vy + gravity) * friction
  vx *= friction
  x += vx
  y += vy
  ```
- **碰撞检测**：粒子碰到画布底部时发生弹性碰撞，速度衰减为原来的30%

#### 粒子消亡
- **生命周期**：每个粒子有30-60帧的生命周期，生命结束自动消失
- **边界检测**：粒子超出画布边界时直接移除
- **落地逻辑**：粒子速度过低时停止运动，绘制到画布图像上并从粒子列表中移除

### 2.2 不同画笔工具的绘制算法差异

| 画笔类型 | 绘制算法特点 | 粒子效果 |
|----------|--------------|----------|
| **手指(FINGER)** | 径向渐变圆形绘制，中心完全不透明渐变到边缘完全透明 | 生成3个粒子，速度1-3 |
| **手掌(PALM)** | 椭圆形径向渐变，尺寸是手指画笔的2倍宽，1.3倍高 | 生成8个粒子，速度1-3 |
| **拳头(FIST)** | 圆形渐变，尺寸是手指的1.5倍，过渡更硬 | 生成10个"强力"粒子，速度2-6 |
| **耙子(RAKE)** | 5个等距分布的小圆点，横向间距15px | 生成5个粒子 |
| **梳子(COMB)** | 9个等距分布的竖线，横向间距6px | 生成3个粒子 |
| **喷洒(SPRAY)** | 25个随机分布的小点，在2倍画笔大小范围内随机分布 | 生成15个喷洒粒子，速度3-8 |
| **印章(STAMP)** | 根据印章图案(圆形、星形、心形、叶形)绘制实心形状 | 生成8个粒子 |
| **添沙(ERASER)** | 高斯分布的随机点，添加沙子颜色到画布 | 不生成粒子 |

## 3. 性能考量

### 3.1 现有性能优化措施

1. **数据结构优化**：
   - `SandParticle`使用`@dataclass(slots=True)`，减少内存占用和属性访问开销
   - 粒子列表更新采用单次遍历+原地过滤，避免频繁的列表删除操作

2. **渲染优化**：
   - 粒子绘制按颜色分组，减少QPainter的setBrush调用次数，降低状态切换开销
   - 避免重复调用`update()`，仅在必要时触发重绘
   - 鼠标移动绘制采用插值算法，根据距离自动调整绘制步长，避免过度绘制

3. **资源限制**：
   - 粒子数量上限500，防止粒子爆炸导致性能下降
   - 历史记录最多保存30步，限制内存使用

4. **算法优化**：
   - 物理计算直接使用局部变量缓存属性，减少属性查找开销
   - 绘制操作尽可能复用QPainter对象，避免频繁创建销毁

### 3.2 性能瓶颈与优化建议

#### 现有瓶颈
1. **粒子数量限制**：当前上限500粒子，若要支持数千粒子，Python层面的循环计算会成为瓶颈
2. **单线程处理**：所有绘制和物理计算都在主线程执行，粒子数量增加会导致UI卡顿
3. **绘制性能**：QPainter的CPU绘制在高分辨率下性能有限，大量粒子绘制会占用过多CPU资源
4. **内存使用**：历史记录保存完整的QImage副本，30步记录会占用较多内存

#### 优化建议
1. **GPU加速**：
   - 使用QOpenGLWidget替代QWidget，利用OpenGL进行粒子绘制和物理计算
   - 将粒子数据存储在GPU显存中，使用着色器实现粒子运动和渲染

2. **多线程优化**：
   - 将粒子物理计算移到独立工作线程，主线程仅负责渲染
   - 采用对象池技术复用粒子对象，避免频繁创建销毁

3. **算法优化**：
   - 使用numpy向量化计算替代Python循环，大幅提升粒子物理更新速度
   - 实现空间分区算法，减少碰撞检测的计算量

4. **内存优化**：
   - 历史记录只保存差异数据，而非完整图像副本
   - 采用压缩算法压缩历史图像数据

## 4. 扩展性评估

### 4.1 新增「水波纹」画笔效果的修改模块

要新增水波纹画笔效果，需要修改以下模块：

1. **模型层**：
   - 在`brush.py`的`BrushType`枚举中添加`RIPPLE = "水波纹"`类型

2. **视图层**：
   - 在`main_window.py`的工具列表中添加水波纹画笔按钮
   - 在`canvas.py`中添加`_draw_ripple`方法，实现水波纹绘制算法
   - 在`draw_at`方法中添加水波纹画笔的分支处理
   - 在`_draw_brush_preview`方法中添加水波纹的预览效果

3. **粒子系统扩展**：
   - 可以扩展`SandParticle`添加波动相关属性，或新增独立的`RippleParticle`类
   - 在`_update_particles`中添加水波纹传播的物理计算逻辑

### 4.2 参数校验机制与扩展方式

#### 现有校验机制设计
参数校验采用**集中式校验+属性访问器**的设计模式：
1. **集中式校验函数**：在`validators.py`中定义统一的校验函数，每个校验函数负责一种类型的参数验证，包括类型检查、范围检查、自动修正和错误日志记录
2. **异常处理**：自定义`ValidationError`异常，统一处理校验失败场景
3. **常量定义**：所有参数的有效范围都定义为常量，便于统一维护
4. **属性绑定**：在`SandCanvas`类中使用`@property`装饰器，在属性设置时自动调用校验函数，确保参数始终有效

#### 添加新校验规则的方法
1. **定义常量**：在`validators.py`的常量定义区域添加新参数的有效范围
2. **实现校验函数**：添加对应的校验函数，遵循统一的函数签名和错误处理规范
3. **绑定到属性**：在需要校验的类中使用`@property`装饰器，在setter中调用新增的校验函数
4. **全局应用**：在所有设置该参数的地方调用校验函数，确保参数有效性

示例：添加水波纹振幅校验
```python
# 1. 添加常量
RIPPLE_AMPLITUDE_MIN = 0.0
RIPPLE_AMPLITUDE_MAX = 10.0

# 2. 添加校验函数
def validate_ripple_amplitude(amplitude: float) -> float:
    if not isinstance(amplitude, (int, float)):
        raise ValidationError(f"水波纹振幅必须是数字，收到: {type(amplitude).__name__}")
    amplitude = float(amplitude)
    return max(RIPPLE_AMPLITUDE_MIN, min(RIPPLE_AMPLITUDE_MAX, amplitude))

# 3. 绑定到属性
@property
def ripple_amplitude(self) -> float:
    return self._ripple_amplitude

@ripple_amplitude.setter
def ripple_amplitude(self, value: float):
    self._ripple_amplitude = validate_ripple_amplitude(value)
```
