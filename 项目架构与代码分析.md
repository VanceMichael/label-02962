# 沙画台应用程序 - 架构与代码分析

## 1. 架构分析

### 1.1 分层架构设计

项目采用了清晰的**四层分层架构**，各层职责明确：

```
backend/
├── models/          # 数据模型层 - 业务数据与核心类型
├── utils/           # 工具层 - 通用功能与辅助函数
├── widgets/         # 组件层 - 自定义UI组件
└── windows/         # 窗口层 - 主窗口与页面逻辑
```

#### 各层职责详解：

| 层级 | 职责 | 核心文件 |
|------|------|----------|
| **models/** | 定义业务数据模型和核心类型枚举，不包含UI逻辑 | `particle.py` - 粒子数据类<br>`brush.py` - 画笔类型枚举 |
| **utils/** | 提供通用工具函数，包括参数校验、日志配置等，与业务逻辑解耦 | `validators.py` - 参数校验模块<br>`logger.py` - 日志配置 |
| **widgets/** | 自定义UI组件，封装特定功能的交互逻辑 | `canvas.py` - 沙画画布核心组件<br>`tool_button.py` - 工具按钮 |
| **windows/** | 主窗口和页面级逻辑，负责UI布局、事件分发和业务协调 | `main_window.py` - 主窗口 |

**架构优点**：
- 单一职责原则：每层只负责特定功能
- 依赖方向清晰：上层依赖下层，下层不依赖上层
- 可测试性：工具层和模型层可独立测试
- 可扩展性：新增功能不会破坏现有架构

### 1.2 画布组件（Canvas）实现原理

## 2. 核心算法

### 2.1 粒子系统实现原理

粒子系统位于 `canvas.py:495-577`，是沙画效果的灵魂。

#### 粒子数据结构 (`models/particle.py:9`)：

```python
@dataclass(slots=True)  # 使用 __slots__ 优化内存访问
class SandParticle:
    x: float          # 位置X
    y: float          # 位置Y
    vx: float         # 速度X
    vy: float         # 速度Y
    size: float       # 粒子大小
    color: QColor     # 粒子颜色
    life: int         # 剩余生命周期（帧数）
```

**设计亮点**：使用 `@dataclass(slots=True)` 减少内存占用并提高访问速度。

#### 粒子生命周期：

##### 1. 生成阶段 (`_spawn_particles()`:497)

```python
def _spawn_particles(self, pt: QPoint, count: int, strong: bool = False, spray: bool = False):
    # 数量上限保护
    count = min(count, self.max_particles - len(self.particles))
    
    for _ in range(count):
        angle = random.uniform(0, 2 * math.pi)
        speed = random.uniform(1, 3)  # 普通速度
        if strong:
            speed = random.uniform(2, 6)  # 强力模式（拳头）
        if spray:
            speed = random.uniform(3, 8)  # 喷洒模式
        
        # 颜色添加随机噪点
        noise = random.randint(-10, 10)
        
        self.particles.append(SandParticle(
            x=px + random.uniform(-5, 5),
            y=py + random.uniform(-5, 5),
            vx=speed * math.cos(angle),
            vy=speed * math.sin(angle) - random.uniform(1, 3),  # 初始向上抛
            size=random.uniform(1, 3),
            color=QColor(..., 220),  # 半透明
            life=random.randint(30, 60)  # 30-60帧生命周期
        ))
```

**生成策略**：
- 随机角度散射：`0 ~ 2π` 均匀分布
- 速度分层：普通/强力/喷洒三种速度模式
- 初始上抛：`vy` 减去 `1~3`，模拟抛洒效果
- 颜色噪点：±10 变化，模拟真实沙子纹理

##### 2. 运动更新 (`_update_particles()`:539)

**物理更新算法**（每帧执行）：

```python
for p in self.particles:
    # 1. 物理模拟
    p.vy = (p.vy + gravity) * friction  # 重力 + 垂直摩擦
    p.vx *= friction                    # 水平摩擦
    p.x += p.vx                         # 位置更新
    p.y += p.vy
    p.life -= 1                         # 生命衰减
    
    # 2. 边界检测
    if p.x < 0 or p.x > w or p.life <= 0:
        continue  # 出界或死亡，不加入存活列表
    
    # 3. 底部碰撞
    if p.y >= h - p.size:
        p.y = h - p.size
        p.vy *= -0.3  # 能量损失反弹
        p.vx *= 0.8   # 水平摩擦
        
        if abs(p.vy) < 0.5:
            # 速度足够小，落地融合到画布
            canvas.setPixelColor(px, int(p.y), p.color)
            continue  # 不保留在粒子列表中
    
    alive.append(p)
```

**物理模型特点**：
- 重力叠加：每帧 `vy += gravity`
- 摩擦衰减：速度乘以摩擦系数 `0.95`
- 碰撞反弹：保留 `30%` 垂直速度，`80%` 水平速度
- 落地融合：速度小于阈值时，粒子永久绘制到画布

##### 3. 消亡条件

粒子消亡的三种途径：
1. **生命周期结束**：`life <= 0`（30-60帧）
2. **出界**：水平方向超出画布
3. **落地融合**：底部碰撞后速度足够小，永久绘制到画布

#### 性能优化要点：

- **单次遍历过滤**：使用单次遍历 + 新建列表的方式，避免多次删除操作（O(n) 复杂度）
- **局部变量缓存**：`gravity`, `friction`, `w`, `h` 等属性缓存到局部变量，减少属性查找开销
- **粒子上限**：`max_particles = 500` 防止无限增长

## 3. 性能考量

### 3.1 现有性能优化措施

#### 1. 粒子系统优化

**措施1：粒子数量上限**
```python
# canvas.py:62
self.max_particles = 500  # 硬上限防止性能雪崩
```

**措施2：按颜色分组批量绘制** (`paintEvent()`:235-249)
```python
# 按颜色 rgba 分组
color_groups: dict[int, list[SandParticle]] = {}
for p in self.particles:
    key = p.color.rgba()
    color_groups[key].append(p)

# 每组设置一次画刷，批量绘制同色粒子
for rgba, group in color_groups.items():
    painter.setBrush(QBrush(group[0].color))
    for p in group:
        painter.drawEllipse(...)
```
**原理**：减少 QPainter 状态切换次数。状态切换（setBrush）是昂贵操作，从 O(n) 减少到 O(k)，k 为颜色种类数。

**措施3：单次遍历过滤** (`_update_particles()`:539)
```python
# 优化前（潜在问题）：多次 remove() 导致 O(n²)
# 优化后：单次遍历新建列表
alive = []
for p in self.particles:
    if should_keep:
        alive.append(p)
self.particles = alive
```
**复杂度**：O(n) → O(n)，但常数因子大幅降低。

```python

```
**收益**：
- 减少内存开销（每个对象节省 `__dict__` 指针）

### 3.2 潜在性能瓶颈与优化建议

#### 瓶颈1：粒子数量限制（当前 500）

**问题现象**：
- 快速绘制时粒子可能达到上限，新粒子不再生成
- 视觉效果可能出现"断流"

**瓶颈分析**：
- 每粒子更新：约 20 次浮点运算 + 条件判断
- 每粒子绘制：QPainter.drawEllipse() 是 GPU 绘制调用
- 500 粒子 × 33 FPS = 16,500 次/秒

**优化建议**：

| 优化方案 | 预期收益 | 实现复杂度 |
|---------|----------|------------|
| **GPU 粒子系统** | 提升到 5000+ 粒子 | 高 |
| **粒子池化** | 减少 GC 开销 30% | 中 |
| **空间分区更新** | 减少 20-40% 更新量 | 中 |

**方案1：GPU 粒子系统**
使用 QOpenGLWidget 替代 QWidget，在着色器中更新粒子：
```glsl
// 顶点着色器伪代码

in vec2 velocity;
in float life;

uniform float gravity;
uniform float deltaTime;

void main() {
    velocity.y += gravity * deltaTime;
    position += velocity * deltaTime;
    life -= 1.0;
    // ...
}
```

**方案2：粒子对象池**
```python
class ParticlePool:
    def __init__(self, max_size=1000):
        self.pool = [SandParticle(0,0,0,0,0,QColor(),0) 
                     for _ in range(max_size)]
```

```python
def _draw_water(self, painter: QPainter, pt: QPoint):

    # 绘制3-5层同心圆
    for i in range(5):

        width = max(1, int(self.brush_size * 0.1))
```

### 4.2 参数校验机制分析

#### 校验机制架构

校验模块位于 `backend/utils/validators.py`，采用**函数式校验**设计。

##### 核心设计模式：

```
参数输入 → 类型校验 → 范围校验 → 自动修正 → 返回结果
                    ↓
                日志记录
                    ↓
          （异常情况抛出 ValidationError）
```

##### 校验函数结构（以 `validate_brush_size()` 为例）：

```python
def validate_brush_size(size: int) -> int:
    # 1. 类型校验
    if not isinstance(size, (int, float)):
        logger.error(f"类型错误: {type(size)}")
        raise ValidationError(f"必须是数字，收到: {type(size).__name__}")
    
    # 2. 类型转换
    size = int(size)
    
    # 3. 范围校验 + 自动修正（软性失败）
    if size < BRUSH_SIZE_MIN:
        logger.warning(f"小于最小值 {BRUSH_SIZE_MIN}，已修正")
        return BRUSH_SIZE_MIN
    if size > BRUSH_SIZE_MAX:
        logger.warning(f"大于最大值 {BRUSH_SIZE_MAX}，已修正")
        return BRUSH_SIZE_MAX
    
    # 4. 返回校验后的值
    return size
```

#### 校验策略分类：

| 策略类型 | 处理方式 | 适用场景 | 示例 |
|---------|----------|----------|------|
| **硬性失败** | 抛出 `ValidationError` | 类型错误、格式错误 | 颜色字符串格式不对 |
| **软性修正** | 日志警告 + 返回边界值 | 数值超出范围 | 画笔大小 150 → 修正为 100 |
| **静默通过** | 仅日志记录 | 非关键参数 | 文件扩展名不匹配 |
